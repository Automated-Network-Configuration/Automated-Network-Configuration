Network Switches: Configuration Guide

Understanding switches, avoiding common pitfalls, and preventing network disasters

What Are Network Switches?

Network switches are fundamental networking devices that operate at the Data Link Layer (Layer 2) of the OSI model. They create a network by connecting multiple devices and enabling communication between them within a local area network (LAN).

How Switches Work

  • Learn MAC addresses of connected devices
  • Build and maintain MAC address tables
  • Forward frames based on destination MAC addresses
  • Create separate collision domains for each port
  • Provide full-duplex communication

Common Switch Uses

  • Connecting computers, servers, and printers in offices
  • Creating network segments in data centers
  • Linking access points in wireless networks
  • Connecting IoT devices in smart buildings
  • Building redundant network paths for reliability
  • Implementing VLANs for network segmentation

Automated Switch Configuration

Modern network management increasingly relies on automation to ensure consistent, error-free configurations across multiple switches. Using Python scripts and YAML configuration files provides a reliable, repeatable way to deploy switch configurations.

Why Automate Switch Configuration?

  • Eliminates human error in repetitive configuration tasks
  • Ensures consistency across all switches in the network
  • Enables version control for network configurations
  • Speeds up deployment and reduces configuration time
  • Facilitates disaster recovery and switch replacement
  • Provides audit trails and documentation

The Automation Workflow

  • Step 1: Define switch configurations in YAML files (human-readable format)
  • Step 2: Python script reads and validates the YAML configuration
  • Step 3: Script connects to switches via SSH
  • Step 4: Commands are generated and applied to the switch
  • Step 5: Configuration is verified and results are logged

Python Configuration Script for Switch 1

import yaml
from netmiko import ConnectHandler

# Define device details
cisco_switch = {
    "device_type": "cisco_ios",
    "ip": "192.168.1.100",   # Replace with switch management IP
    "username": "admin",     # Replace with your username
    "password": "adminpass", # Replace with your password
    "secret": "StrongPassword123",  # Enable secret
}

# Configuration commands
config_commands = [
    "hostname Switch1",
    "enable secret StrongPassword123",
    "banner motd #Unauthorized access is prohibited!#",
    "interface vlan 1",
    " ip address 192.168.1.2 255.255.255.0",
    " no shutdown",
    "ip default-gateway 192.168.1.1",
    "line console 0",
    " password ciscocon",
    " login",
    " logging synchronous",
    "line vty 0 4",
    " password ciscovty",
    " login",
    " transport input telnet ssh",
    "interface range FastEthernet0/10 - 24",
    " shutdown",
    "end",
    "write memory",
]

def main():
    # Connect to device
    print("Connecting to switch...")
    connection = ConnectHandler(**cisco_switch)
    
    # Enter enable mode
    connection.enable()
    
    # Send configuration commands
    print("Sending configuration...")
    output = connection.send_config_set(config_commands)
    print(output)
    
    # Save configuration
    print("Saving configuration...")
    save_output = connection.save_config()
    print(save_output)
    
    # Disconnect
    connection.disconnect()
    print("Configuration complete and connection closed.")

if __name__ == "__main__":
    main()

YAML Configuration File for Switch 1

switches:
  ---
- name: Configure Cisco Switch with SSH
  hosts: switches
  gather_facts: no
  connection: network_cli

  tasks:
    - name: Set hostname
      cisco.ios.ios_config:
        lines:
          - hostname Switch1

    - name: Configure enable secret
      cisco.ios.ios_config:
        lines:
          - enable secret StrongPassword123

    - name: Configure MOTD banner
      cisco.ios.ios_config:
        lines:
          - banner motd #Unauthorized access is prohibited!#

    - name: Create VLANs
      cisco.ios.ios_config:
        lines:
          - vlan 10
          - name Sales
          - vlan 20
          - name HR
          - vlan 30
          - name IT

    - name: Configure management VLAN IP
      cisco.ios.ios_config:
        lines:
          - ip address 192.168.1.2 255.255.255.0
          - no shutdown
        parents: interface vlan 1

    - name: Configure default gateway
      cisco.ios.ios_config:
        lines:
          - ip default-gateway 192.168.1.1

    - name: Assign VLAN 10 to Fa0/1
      cisco.ios.ios_config:
        lines:
          - switchport mode access
          - switchport access vlan 10
        parents: interface FastEthernet0/1

    - name: Assign VLAN 20 to Fa0/2
      cisco.ios.ios_config:
        lines:
          - switchport mode access
          - switchport access vlan 20
        parents: interface FastEthernet0/2

    - name: Assign VLAN 30 to Fa0/3
      cisco.ios.ios_config:
        lines:
          - switchport mode access
          - switchport access vlan 30
        parents: interface FastEthernet0/3

    - name: Configure console line
      cisco.ios.ios_config:
        lines:
          - password ciscocon
          - login
          - logging synchronous
        parents: line console 0

    - name: Set domain name (required for SSH)
      cisco.ios.ios_config:
        lines:
          - ip domain-name example.local

    - name: Generate RSA keys for SSH
      cisco.ios.ios_command:
        commands:
          - crypto key generate rsa modulus 2048

    - name: Create local user for SSH login
      cisco.ios.ios_config:
        lines:
          - username admin privilege 15 secret AdminPass123

    - name: Configure VTY lines for SSH only
      cisco.ios.ios_config:
        lines:
          - login local
          - transport input ssh
        parents: line vty 0 4

    - name: Shutdown unused ports (Fa0/4 - Fa0/24)
      cisco.ios.ios_config:
        lines:
          - shutdown
        parents: interface range FastEthernet0/4 - 24

    - name: Save configuration
      cisco.ios.ios_config:
        save_when: always

Critical Reminder

Network switch misconfigurations can have cascading effects throughout your entire infrastructure. A single misconfigured switch can compromise the security of hundreds or thousands of connected devices. Always test configurations in a lab environment before deploying to production, and maintain comprehensive documentation of all network changes.
